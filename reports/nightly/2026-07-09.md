## Phase 3: What I Built

### Shareable Valuation URLs — `/valuation/[artistId]`

**New files:**
- `app/valuation/[artistId]/page.tsx` — Dynamic route with `generateMetadata()` for artist-specific OG tags (title, description, artist image). Server component fetches artist info from Spotify via Recoup API.
- `app/api/spotify/artist/[id]/route.ts` — API route to fetch a single Spotify artist by ID (proxies to Recoup API)
- `lib/spotify/fetchArtist.ts` — Client-side artist fetch for auto-select on shareable URLs
- `lib/spotify/fetchArtistServer.ts` — Server-side fetch with 24h revalidation cache for OG metadata

**Modified files:**
- `hooks/useCatalogValuation.ts` — Accepts optional `initialArtistId` param. When provided, auto-fetches artist info, auto-selects it, and either auto-runs (if authenticated) or queues for Privy login then auto-runs.
- `components/valuation/CatalogValuation.tsx` — Accepts and passes `initialArtistId` prop
- `components/valuation/ShareValuation.tsx` — Share URLs now use `/valuation/${artistId}` instead of generic `/valuation`
- `components/valuation/ValuationResult.tsx` — Passes `artistId` to ShareValuation

**The full flow:**
1. User runs valuation → result card share buttons now use artist-specific URL
2. User shares on X → tweet includes `recoupable.dev/valuation/4q3ewBCu7t80kXyVo...`
3. LinkedIn/X render OG preview: "What's Bad Bunny's Catalog Worth? | Recoup" with artist image
4. Recipient clicks → lands on dynamic route → auto-fetches artist → Privy gate → auto-runs valuation
5. Recipient sees result → tries their own artist → viral loop complete
