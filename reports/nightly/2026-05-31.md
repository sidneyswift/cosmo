## Phase 3: What I Built

### 1. Fixed AI Gateway Model ID (Root Cause #1)

**The bug:** `ai-gateway.js` defaulted to `anthropic/claude-sonnet-4.6-20250725` — a model that no longer exists on the Vercel AI Gateway. Every `callLLM()` call returned a `model_not_found` error, which was caught and returned as `null`. The extraction phase reported "Total moments extracted: 0" and exited cleanly — no error, no crash, just silent failure.

**Fix:** Updated default model to `anthropic/claude-sonnet-4-20250514` in `scripts/lib/ai-gateway.js`.

**Impact:** This broke ALL content engine scripts that use `callLLM()` — not just brain-first-generate. The nightly engine, QA judge, critic, and any script using the shared gateway were all silently failing.

### 2. Fixed OOM in brain-first-generate.js (Root Cause #2)

**The bug:** Even with working LLM calls, the script would OOM during post generation because:
- Reading 200 meeting files (each up to 4KB) = ~800KB of transcript data in memory
- Batching 15 meetings per extraction call = large prompt payloads
- 3000-char transcripts per meeting = unnecessary detail for moment extraction
- All data accumulated across the full run with no GC pressure

**Fixes (4 changes to brain-first-generate.js):**
1. Reduced meeting file scan from `head -200` to `head -50` (most recent = most relevant)
2. Reduced transcript slice from 3000 → 1500 chars per meeting
3. Reduced extraction batch size from 15 → 5 meetings per LLM call
4. Added explicit GC hint between post generation iterations
5. Reduced default LIMIT from 10 → 5 posts per run (can override with --limit=N)

**Also created:** `scripts/run-brain-first.sh` wrapper that sets `NODE_OPTIONS="--max-old-space-size=512"` for safety.
