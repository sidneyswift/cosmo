## Phase 3: What I Built

### 1. New Agent Tool: `get_shelf_books.ts` (120 lines)

**File:** `apps/web/agent/tools/get_shelf_books.ts`

A family-scoped read-only tool that queries the `shelf_books` table. Follows the established `get_children.ts` pattern exactly:

- **Both access modes:** Caller path (web chat, uses family's Supabase JWT + RLS) and pinned path (SMS/dispatch/agent-job, uses `getShelfBooksPinned` direct query)
- **Optional text search:** `search` parameter does ilike across title, author, and notes — so asking about "math" finds all math books
- **Returns:** id, title, author, coverUrl, tocUrl, notes for each active book
- **Smart empty states:** Different messages for "no books match your search" vs "family hasn't added any books yet"
- **Bounded:** `TOOL_DEADLINE.quick`, repeatable, max 50 results (default 20)

### 2. Pinned Path: `getShelfBooksPinned` in `agent-pinned.ts` (74 lines)

For sessions without a caller JWT (SMS channel, email, dispatch, agent-job, materials-lab) — the same trust basis as every other pinned tool. Direct Postgres query with family_id scoping, optional ilike search, ordered by created_at desc.
