## Content Pipeline Quality Overhaul (9:30 PM)
Sid asked me to audit all 27 approved posts against the copywriting skill. Results were bad:
- Deleted 19 of 27 posts (15 initial + 4 borderlines)
- 6 posts all had the same thesis ("backend ops > flashy AI creation")
- Many had 3-5 negative parallelism violations the judge didn't catch
- Posts were on-voice but had zero actionable value

### Root Cause
Three gaps in qa-judge.js:
1. No thesis-level dedup (only idea-level dedup existed in dedup.js)
2. Anti-slop was a soft fail — judge would try to rewrite instead of rejecting
3. No "value test" — judge checked voice/safety/fit but not whether reader learns anything

### Fixes Shipped (all in qa-judge.js)
1. **Thesis Dedup Gate** — `checkThesisDedup()` runs BEFORE the judge. Fetches last 30 approved/published drafts, uses LLM to compare core thesis. Duplicate thesis = instant reject.
2. **Hard Anti-Slop Enforcement** — New fields: `anti_slop_count`, `anti_slop_in_hook`, `anti_slop_details`. 3+ violations = hard reject. Hook uses banned pattern = hard reject. No rewrite attempts.
3. **Value Gate** — New field: `value_score` (1-10). Score < 5 = hard reject. Questions: "Would this be worth reading without Sid's name?" / "Can the reader DO something after reading?"

Also added 3 new learned rules to guardrails.json encoding these lessons permanently.

8 approved posts remain after the purge.

## Content Pipeline Architecture Overhaul (10 PM)
Three more structural changes beyond the QA judge fixes:

### 1. Killed External Sources (topic-sourcer.js)
Disabled X High Signal List and Industry News fetching. Data showed:
- `manual` ideas = 87% kept, `brain-first` = 56% kept
- `x_search` = 1.5% kept, `reddit` = 0%, `rss` = 3%
Topic sourcer is now brain-only mode — reads meetings, decisions, concepts, and memory files.

### 2. Simplified Writer Prompt (generate.js)
Old generateDraftFromPacket() had 3000+ words: proven formula analytics, competitive intelligence, bookmarkability rubrics, voice profiles, winning formula data, content mix targets, audience positioning. Every post came out sounding the same — LLM was pattern-matching to a template.

New prompt is ~200 words: here's what happened, here's what Sid thinks, write it like he'd say it to a friend. Anti-slop rules still in the system prompt via callOracle().

### 3. High-Angle Fast Path + Postmortem Removal (generate.js)
Old pipeline: write → critique → revise → critique → revise → postmortem → final revision (7 LLM calls minimum).
New pipeline:
- If first critic pass has voice >= 8 and score >= 6: SKIP all revisions. Already sounds like Sid.
- Still checks for hard fails (fabricated data, 3+ slop flags)
- Removed viralPostmortem() entirely — it was adding a revision pass on every passing draft, smoothing out personality.
- Standard path (voice < 8 or score < 6): still does up to MAX_REVISIONS critique/revise loops, but with a simplified revision prompt.

All 3 files parse clean (node --input-type=module --check).

Old pipeline: context packet (1) + write (1) + tighten (0-1) + critique (1) + revise (1) + critique (1) + revise (1) + postmortem (1) + final revision (0-1) = **7-8 calls**
New pipeline (high-angle): write (1) + critique (1) = **2 calls**
New pipeline (standard): write (1) + critique (1) + revise (1) + critique (1) = **4 calls**

### Net Token Savings
- Removed: context packet LLM call, postmortem, final revision, 3000-word prompt → 200 words
- Estimated: 60-70% fewer tokens per draft
