### Bug #7 Found: qa-judge.js import-time execution
`qa-judge.js` had `main().catch(...)` at module bottom — runs unconditionally. When nightly-engine.js imports `{ judgeAllReady }`, that `main()` kicks off an unawaited async promise. It races against generate and can reject freshly-created drafts before `judgeAllReady()` gets to them.

## Content Engine Voice Fix (6:30 PM)

### Problem Sid Identified
Said "Fix" — drafts still read like observer commentary, not builder voice. Only 1/9 passing QA.

### Root Cause: Anti-Fabrication Rules Were Killing Sid's Voice
The writer prompt had ABSOLUTE anti-fabrication rules that banned first-person voice:
- "NEVER use I, we, our, my followed by a specific action"
- "Write OPINIONS and ANALYSIS only"
- Brain context was labeled "[ANONYMIZED — strip all names/companies/numbers]" — even Recoup!

This made it impossible to write AS Sid. Every draft came out as generic commentator.

### Fix: Rewrote Writer Prompt from Commentator → Builder
1. System prompt: "Write AS Sid, not ABOUT Sid" — first person expected, not banned
2. Anti-fabrication → Voice rules: ground first-person in brain context, don't ban it
3. Brain context: Recoup experiences no longer anonymized (it's his own company)
4. Audience positioning: "builder, not commentator"
5. Instructions: "BUILDER FORMAT" replaces "OPINION FORMAT"

### Results
- Before: 1/9 passed QA (11%), avg score 6.1, observer voice
- After: 12/24 passed QA (50%), avg score 7.1, builder voice
- Top hooks: "I skipped the advisor paperwork" (9.4), "I became the code review bottleneck" (8.4)
- Every draft now reads like Sid talking from experience

### Key Lesson
The over-correction against fabrication killed authenticity. The right balance: allow first-person grounded in brain context, don't invent specific numbers/anecdotes, but DO write from builder perspective. Trust the brain to provide real experiences.
