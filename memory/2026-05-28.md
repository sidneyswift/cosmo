## Nightly Value Build: approve.js Fix + Content Engine (1:00 AM)

### What I Built
Fixed the approve.js thumbnail bug that's been haunting us since May 21. Two root causes:
1. No validation gate — any approved draft published regardless of media
2. Supabase URL handling broken — `media_urls` comes as pre-parsed array from Supabase adapter, but code assumed JSON string; also, auto-slides uses slug-named dirs not `draft-{id}` dirs

Three changes to approve.js:
- Thumbnail gate: require valid media before publishing, else send back to 'ready'
- Supabase URL pipeline: download → re-upload to Postiz
- Polymorphic parsing: handle both array and string `media_urls`

### Content Engine Results
- 16 new ideas sourced

- Draft #922 "We said 'this feature will find PMF' before and moved on" — published to LinkedIn with thumbnail
- 3 ready drafts with valid thumbnails (#906, #800, #678) waiting for Sid's approval on dashboard

### Key Lessons
- Supabase adapter returns `media_urls` as parsed arrays, not JSON strings — any code doing `JSON.parse(draft.media_urls)` will break silently
- auto-slides stores thumbnails at slug-based paths in Supabase, not `draft-{id}` directories — approve.js was looking in the wrong place
- The two-adapter pattern (SQLite local + Supabase mirror) means every field access needs to handle both serialized and deserialized forms

### v2: Event-Driven Refactor (7:40 PM)
Sid's feedback: don't run nightly batch → run on-demand when a post gets approved. Always keep one in the queue.

**Architecture change:**
- `slack-review.js` completely rewritten with 3 commands: `post-best`, `watch`, `replenish`
- `watch` is now event-driven: after 👍 → schedule via Postiz → immediately run mini-pipeline (generate 1 → QA judge → thumbnail → post to Slack)
- After 👎 → auto-replenish queue too (always something waiting)
- After thread feedback → regenerate → re-QA → re-thumbnail → re-post revised version
- Empty queue auto-detected → replenish triggers automatically
- `replenish` = new subcommand that runs the mini-pipeline standalone
- `post-best` now checks for pending reviews first (no double-posting)
- Nightly engine no longer auto-schedules → just generates stock and posts best to Slack
- Morning-post cron is now just a seed/fallback

**Result:** The whole content loop is self-sustaining. Sid 👍’s a post, it schedules, and immediately there’s another one waiting. No batches, no waiting until tomorrow.
