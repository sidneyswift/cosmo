## Autonomous Content Engine — End-to-End Automation (9:40 PM)

### What Sid Asked
Fully autonomous nightly content loop: discovery → triage → draft → review → media → schedule. Plus morning reports, performance tracking, and self-improvement.

### What I Built
4 new scripts in content-engine/scripts/:

1. **nightly-engine.js** — Full orchestrator. Runs at 1 AM ET. Does: pulse discovery → sync to Supabase → self-triage (applies Sid's taste patterns automatically) → auto-POV generation → draft generation → self-review (checks for markdown, fabrication, reframe shapes, etc.) → thumbnail generation → schedule 1 post on next available day.

2. **morning-report.js** — Posts to #cosmo-content at 8 AM ET. Shows: yesterday's posts, performance metrics (views/engagement), pipeline stats, nightly engine results, and what was improved. Replaces daily-digest.js.

3. **perf-scraper.js** — Runs at 7 AM ET. Scrapes Sid's LinkedIn via ScrapeCreators API, fuzzy-matches to published drafts, stores views/likes/comments/shares/engagement_rate in new `post_performance` SQLite table. Tracks over time.

4. **self-improve.js** — Runs at 11 PM ET. Analyzes performance data, makes one improvement per run (update hook patterns, log rejection patterns, analyze source quality). Logs all changes to data/improvements.log.

### Updated Files
- **cron-daemon.js** — New schedule: 1AM nightly-engine, 7AM perf-scraper, 8AM morning-report, 11PM self-improve. Kept: 6AM pulse, every-2-min regenerate/auto-slides, 8PM perf-check.
- **auto-slides.js** — Added `cleanTitleForThumbnail()` to strip names/handles/URLs from thumbnail titles (Sid's feedback: don't put other people's names in thumbnails)
- **HEARTBEAT.md** — Added content engine health check and dashboard feedback check tasks
- **package.json** — Added npm scripts for all 4 new scripts

### Self-Triage Logic
Encoded Sid's taste from 2 weeks of feedback:
- Keep: CyrilXBT, Thariq, Tobi Lutke, Karpathy content about agents/AI/building
- Skip: Sam Altman personal, crypto, Reddit memes, hiring, political, aggregator link-only, HN without substance
- Threshold: combined_score > 2.0 minimum, then pattern matching

### Key Architecture Decision
The nightly engine is a single orchestrator that calls existing scripts (pulse, generate, auto-slides, approve, sync) rather than duplicating their logic. This means improvements to any individual script benefit the whole pipeline.
