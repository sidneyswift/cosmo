## Built: recoup-research-plugin (10:34 PM)

Sid said "build something valuable for Recoup." After analyzing the full product landscape — strategy repo, customers, plugins, skills, API, marketplace — identified the biggest gap: the `recoup-research-plugin` was an empty skeleton while research is the #1 entry point for every customer interaction.

### What I built

Populated `recoupable/recoup-research-plugin` from skeleton → fully functional plugin:

**7 skills:**
- `recoup-artist-research` — full artist sweep (the default entry point). Decision tree, synthesis template, all critical gotchas, graceful degradation
- `recoup-playlist-intelligence` — playlist pitching, gap analysis, catalog optimization. Includes the tricky filter flag semantics
- `recoup-audience-analysis` — demographics, TikTok-to-Spotify pipeline, tour routing, market expansion
- `recoup-competitive-analysis` — head-to-head comparison, roster benchmarking, collab finder, release timing
- `recoup-trend-detection` — A&R discovery, chart analysis, viral song autopsy, discovery filtering
- `recoup-people-outreach` — industry people search, contact enrichment, outreach drafting, CRM patterns
- `recoup-web-intelligence` — web search, deep research, URL extraction, entity enrichment, graceful degradation fallback

**4 commands:**
- `/recoup-research [artist]` — full sweep → executive brief
- `/recoup-scout [genre]` — A&R discovery → scouting report
- `/recoup-pitch [artist]` — playlist pitch targets → ranked list
- `/recoup-compare [A] vs [B]` — head-to-head comparison

**2 agents:**
- `research-analyst` — deep synthesis persona
- `market-scout` — trend/discovery persona

**3 reference docs** (ported from `recoupable/skills` music-industry-research skill):
- `endpoints.md` — full curl examples per endpoint, filter semantics, latency budgets
- `response-shapes.md` — actual JSON response structures
- `workflows.md` — 11 multi-step research workflow chains with chaining rules

**4 eval fixtures** for validation.

All three plugin manifests updated (Claude Code, Codex, Cursor). Plugin version 0.1.0.

### Also: Added to marketplace

Updated `recoupable/marketplace` to include the research plugin in both `.claude-plugin/marketplace.json` and `.agents/plugins/marketplace.json`. Customers can now discover and install it via the marketplace.

### Why this matters

- Research is the entry point — every customer interaction starts with "research this artist"
- The monolithic skill in `recoupable/skills` was great but not in the plugin architecture customers install
- Splitting into 7 focused skills means agents route to the right capability faster
- Commands give one-shot workflows (customer types `/recoup-research Drake` and gets a full brief)
- This follows the proven pattern from `recoup-catalog-deals` (the most fleshed-out plugin)
- Directly addresses the "adoption gap" from customers.md — making research Sid-quality without Sid

## Research Plugin Bug Fixes (23:31 ET)

Sid asked me to fix and test the recoup-research-plugin. Found and fixed these verified bugs:

### Bugs Fixed:
1. **Profile field paths (critical)**: Top-level `sp_followers`, `sp_monthly_listeners` etc on `/research/profile` are ALWAYS null. Real data is in `cm_statistics.sp_followers`. Every skill was using wrong paths → agents would produce briefs with null streaming numbers.
2. **Base URL**: Plugin used `recoup-api.vercel.app` but official docs say `api.recoupable.com`. Both work but updated to canonical.
3. **Genre IDs**: Examples used `genre=86` but real IDs are 501xxx+ (e.g. hip-hop/rap=501121). Agents would get 400 errors.
4. **career_stage/recent_momentum**: Not on profile at all — must get from `/research/similar`. Documented.
5. **Discover endpoint**: Returns empty `{ artists: [] }` for any filter combo — documented as known issue with fallback.
6. **Credit costs**: Added documentation for credit-gated endpoints (albums, lookup, festivals, radio = 5 credits each). Agents now know to surface checkout URL instead of failing silently.
7. **jq control chars**: Profile responses contain control chars that break jq — documented workaround.
