## 4. Execution Instructions

When given a topic:

1. Check if topic connects to Recoupable's world (AI infrastructure
   for music, agent capabilities, label operations, artist tools,
   enterprise value, product updates). If not, reject or find the angle
2. Read [[brand-voice]] for company personality
3. Read [[hooks]] and pick the best hook formula
4. Read [[repurpose]] for the production chain order
5. Write for FIRST platform in chain
6. For each subsequent platform, RETHINK — don't reformat
7. Apply [[scheduling]] rules
8. Output one native post per platform, ready to publish

CRITICAL: This is the company talking. "We" not "I".
Confident, not arrogant. Product-focused, not navel-gazing.
Show what the platform does, not how the founder feels about it.

## 5. Content Pillars (Recoupable Brand)

1. **Product capabilities** — what the platform does. demos,
   features, updates. "here's what shipped this week"
2. **Customer results** — how labels and artists use Recoupable.
   case studies, before/after, ROI numbers (with permission)
3. **Industry thought leadership** — how AI is reshaping the
   music business. Recoupable's perspective, backed by data
4. **The thesis** — agents that do what labels do. we prove it
   by running our own. the co-creation model. why context matters
5. **Artist enablement** — how independent artists use AI tools
   to compete. tutorials, workflows, success stories
