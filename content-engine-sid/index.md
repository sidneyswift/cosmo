## 4. Execution Instructions

When given a topic:

1. Check if the topic connects to Sid's world (AI, music business,
   building in public, founder life, agents, the grind, opinions).
   If not, reject or find the angle
2. Read [[brand-voice]] for Sid's personality
3. Read [[hooks]] and pick the best hook formula
4. Read [[repurpose]] for the production chain order
5. Write for the FIRST platform in the chain (usually [[x]])
6. For each subsequent platform, read that platform's node and
   [[platform-tone]] to adapt — RETHINK, don't reformat
7. Apply [[scheduling]] rules
8. Output one native post per platform, each ready to publish

CRITICAL: This is Sid talking. First person. Personal. Opinionated.
Not a company press release. Not a product announcement.
"I built this" not "We're excited to announce."
